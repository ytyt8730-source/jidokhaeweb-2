# ✅ Quality Checklist

> 에이전트와 스킬의 품질 기준 및 검증 항목

## 목적

생성된 에이전트/스킬이 일관된 품질 기준을 충족하도록 보장

---

## 1. 에이전트 품질 기준

### 구조 검증

| 항목 | 기준 | 가중치 |
|------|------|:------:|
| YAML frontmatter | 필수 필드 모두 존재 | 20% |
| name | kebab-case, 고유성 | 10% |
| description | PROACTIVELY 포함 | 10% |
| model | 유효한 모델명 | 10% |
| tools | 유효한 도구 목록 | 10% |

### 내용 검증

| 항목 | 기준 | 가중치 |
|------|------|:------:|
| 역할 정의 | 명확한 한 문장 | 10% |
| 핵심 원칙 | 3-5개 원칙 | 5% |
| When Invoked | 호출 조건 명시 | 10% |
| Process | 단계별 절차 | 10% |
| Provide | 출력 형식 정의 | 5% |

### 통합 검증

| 항목 | 기준 |
|------|------|
| 명칭 충돌 | 기존 에이전트와 중복 없음 |
| 의존성 | 참조 에이전트/스킬 존재 |
| 순환 참조 | A→B→A 패턴 없음 |

### 구조 검증

| 항목 | 기준 | 가중치 |
|------|------|:------:|
| YAML frontmatter | 필수 필드 모두 존재 | 20% |
| name | kebab-case, 고유성 | 10% |

| 항목 | 기준 | 가중치 |

## 3. 점수 계산

### 합격 기준

| 등급 | 점수 | 결과 |
|:----:|:----:|------|
| A | 90-100 | ✅ 즉시 승인 |
| B | 80-89 | ✅ 승인 (권장사항 있음) |
| C | 70-79 | ⚠️ 조건부 승인 |
| D | 60-69 | ❌ 수정 후 재검토 |
| F | <60 | ❌ 거부 |

### 점수 감점 요소

| 요소 | 감점 |
|------|:----:|
| 필수 섹션 누락 | -10 per |
| 예시 부족 | -5 per |
| 불명확한 설명 | -5 |
| 과도한 토큰 | -10 |
| 보안 규칙 위반 | -20 |

## 4. 일관성 검증

### 명명 규칙

```yaml
naming:
  files: kebab-case.md      # 파일명
  agents: kebab-case        # name 필드
  skills: kebab-case        # name 필드
  
examples:
  ✅ code-reviewer.md
  ✅ security-rules
  ❌ CodeReviewer.md
  ❌ security_rules
```

### 출력 형식

```yaml
output_format:
  headers: 마크다운 헤더 (#, ##, ###)
  code: 백틱 코드 블록
  tables: 파이프 테이블
  lists: 불릿 또는 번호
  
consistency:
  이모지: 제목에만 사용
  볼드: **중요 용어**
  코드: `인라인 코드`
```

### 다음 단계 안내

```yaml
next_steps:
  format: "→ @에이전트 요청"
  always_present: true
  
examples:
  ✅ "→ @auditor 검증해줘"
  ✅ "→ @agent-smith 생성해줘"
```

## 5. 품질 체크리스트

### 에이전트 체크리스트

```markdown
## 구조 (40점)
□ [10] name이 kebab-case인가?
□ [10] description에 PROACTIVELY가 있는가?
□ [10] model이 유효한가?
□ [10] tools가 최소 권한인가?

## 내용 (40점)
□ [10] 역할이 명확한가?
□ [10] When Invoked가 구체적인가?
□ [10] Process가 단계별인가?
□ [10] Provide 형식이 정의되었는가?

## 통합 (20점)
□ [10] 명칭 충돌이 없는가?
□ [10] 의존성이 유효한가?

총점: __/100
```

### 스킬 체크리스트

```markdown
## 구조 (35점)
□ [10] name이 kebab-case인가?
□ [10] category가 유효한가?
□ [15] dna가 완전한가?

## 내용 (45점)
□ [10] 목적이 명확한가?
□ [15] 지식이 구조화되었는가?
□ [15] 예시가 2개 이상인가?
□ [5] 제약 조건이 명시되었는가?

## 효율 (20점)
□ [10] 토큰이 1000 이하인가?
□ [10] 줄 수가 150 이하인가?

총점: __/100
```

## 사용 예시

### 예시 1: 에이전트 검증

**입력:** 신규 에이전트 파일

**검증 과정:**
```
구조 검증: 35/40 (tools 과다)
내용 검증: 40/40
통합 검증: 20/20
총점: 95/100 → A등급
```

**결과:** ✅ 승인 (tools 최적화 권장)

### 예시 2: 스킬 검증

**입력:** 신규 스킬 파일

**검증 과정:**
```
구조 검증: 35/35
내용 검증: 30/45 (예시 1개만)
효율 검증: 20/20
총점: 85/100 → B등급
```

**결과:** ✅ 승인 (예시 추가 권장)
