### 2.2 PRD (v1.4) 검토

**참조 기준:** W02-1_PRD 작성하기.pdf

#### ✅ 잘된 점
- 배경 및 맥락이 데이터 기반으로 작성됨 (신규 재참여율 50%)
- 사용자 스토리가 역할별로 체계적
- 성공 지표가 정량/정성 모두 정의됨
- 회원 세그먼트 정의가 명확

#### ❌ 심각한 문제점

**원칙 위반 1: 구체적 구현 세부사항 포함 (금지됨)**

> "PRD는 기술 스택이나 코드 구현 방법(How)은 포함하지 않음" - W02-1, 페이지 4

| 위반 사례 | 섹션 | 조치 |
|----------|------|------|
| "입금자명 형식: MMDD_이름" | §5 신청/결제 | 제거 또는 시스템 구조로 |
| "Framer Motion staggerChildren" | §6 디자인 가이드 | Design System으로 이동 |
| "whileHover={{ y: -4 }}" | §6 애니메이션 사양 | Design System으로 이동 |
| "정기모임: 3일 전 100%, 2일 전 50%" | §5 환불 규정 | 제약 사항으로 이동 (숫자만) |

**원칙 위반 2: 디자인 시스템이 PRD에 포함됨**

§6 UX 및 디자인 가이드에 다음이 포함되어 있음:
- 8px 그리드 시스템
- 폰트 종류 (Pretendard, Noto Serif KR)
- 애니메이션 코드

→ 이 내용은 `design-system.md`와 **중복**됨

#### 📝 권장 조치

```markdown
# PRD에서 제거해야 할 항목

## §5 제품 개념 및 주요 기능
- 입금자명 형식 → 시스템 구조 7장 데이터 요구사항으로
- 응답 대기 시간 (24시간/6시간/2시간) → 시스템 구조 또는 config

## §6 UX 및 디자인 가이드
- 8px 그리드 → design-system.md로 (이미 있음)
- Framer Motion 코드 → design-system.md로 (이미 있음)
- 폰트 종류 → design-system.md로 (이미 있음)
- 애니메이션 사양 전체 → design-system.md로 (이미 있음)

## §5에서 유지할 항목 (제약 사항으로 이동)
- 환불 규정: "정기모임 3일/2일, 토론모임 2주/7일" (수치만)
- 칭찬 제한: "모임당 1명, 3개월 중복 불가"
- 대기 응답 시간: "모임 임박도에 따라 차등"
```

### 2.5 Milestones 검토

**참조 기준:** 전체 문서 + 개발 경험

#### 🔴 심각한 문제: 범위 과다 (Scope Creep)

```
현재 상태:
┌─────────────────────────────────────────────────────────────┐
│  총 67 Phase across 18 Milestones                          │
│  ─────────────────────────────────────────────────────────  │
│  Part 1: MVP (M1~M7) → 30 Phase                            │
│  Part 2: Experience Enhancement (M8~M12) → 18 Phase        │
│  Part 3: 백오피스 MVP (M13~M18) → 19 Phase                  │
└─────────────────────────────────────────────────────────────┘

문제:
- MVP가 M1~M6인데 M7 (Polish & Growth)도 추가됨
- M8~M12는 "Experience Enhancement"로 **2차 개발**인데 같은 문서에 포함
- M13~M18 백오피스는 **3차 개발** 범위
- 총 18개 Milestone은 관리 불가능한 수준
```

#### 📝 권장 조치

**1. 문서 분리**
```
milestones.md
├── milestones-mvp.md      (M1~M6만, 26 Phase)
├── milestones-growth.md   (M7만, 4 Phase)
├── milestones-exp.md      (M8~M12, 18 Phase) - 별도 문서
└── milestones-backoffice.md (M13~M18, 19 Phase) - 별도 문서
```

**2. 현재 milestones.md 간소화**
- Part 1 (M1~M6)만 유지
- M7은 "출시 후 1개월차" 부록으로
- Part 2, Part 3는 별도 파일로 분리

**3. 각 Part별 관리**
- MVP: 현재 집중
- Growth: M6 완료 후
- Experience: MVP 안정화 후 6개월 차
- 백오피스: MVP 안정화 후 2개월차

## 7. 결론

### 7.1 현재 상태 요약

지독해 웹서비스의 문서화 수준은 **일반 스타트업 대비 상위 10% 수준**입니다. 
핵심 가치 정의, 사용자 스토리, 기술 스택 선정 근거 등이 매우 체계적입니다.

그러나 다음 문제로 인해 **AI 에이전트 개발 효율이 저하**될 수 있습니다:
1. 문서 간 중복으로 인한 혼란
2. PRD에 구현 세부사항 포함으로 유연성 저하
3. Milestones 과다 확장으로 집중력 분산

### 7.2 핵심 권장사항

> **"PRD는 What과 Why만, How는 별도 문서에서"** - Modulabs W02-1

1. **서비스 개요**: 본질과 가치에 집중, 기능 상세는 PRD로
2. **PRD**: 구현 코드/UI 제거, 맥락 전달에 집중
3. **Milestones**: MVP (M1~M6) 완료 후 확장 검토
4. **중복 제거**: 하나의 진실의 소스(Single Source of Truth) 원칙
