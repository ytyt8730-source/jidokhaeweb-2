# Milestones 최적화 제안 보고서

**수신:** 지독해 운영자님  
**작성:** AI 아키텍트  
**주제:** "빠른 모임 참가 신청"을 위한 마일스톤 재구성 제안

---

## 1. 개요 및 답변

**"순서가 적절한가요?"**  
→ **아니요, 현재 순서는 '빨리 신청 받는 것'에 최적화되어 있지 않습니다.**  
현재 로드맵에서 가장 빠르게 신청을 받을 수 있는 "계좌이체" 기능이 M5(운영자 도구)에 배치되어 있어, 프로젝트 후반부에야 신청이 가능합니다. 또한 M2의 PG 결제는 심사 기간이 필요하여 오픈을 지연시킬 수 있습니다.

**"수정 제안이 있나요?"**  
→ **네, '선 계좌이체, 후 PG' 전략으로 순서를 변경할 것을 강력히 제안합니다.**  
계좌이체 신청 기능은 구현 즉시 사용 가능하므로 M2로 앞당기고, PG 연동은 M4로 미루면 **오픈 시점을 최소 3~4주 앞당길 수 있습니다.**

---

## 2. 최적화 전략: "계좌이체 선행 (First)"

### 2.1 기존 vs 제안 비교

| 구분 | 기존 로드맵 | 제안 로드맵 (Fast-Track) |
|------|------------|------------------------|
| **M2** | **PG 결제** (포트원 연동, 환불) | **계좌이체 신청** (입금확인, 대기) |
| **신청 가능 시점** | M2 완료 + PG 심사 완료 후 (약 5~6주차) | **M2 완료 즉시 (약 3~4주차)** 🚀 |
| **M3** | 알림 시스템 | 알림 시스템 (동일) |
| **M4** | 소속감 기능 | **PG 결제 연동** + 소속감 (일부) |
| **M5** | **계좌이체** + 운영자 도구 | 소속감 (나머지) + 운영자 도구 |

### 2.2 기대 효과

1.  **조기 오픈:** M2 완료 직후 **"계좌이체 전용 베타 서비스"**를 오픈하여 실제 신청을 받을 수 있습니다.
2.  **리스크 분산:** PG 심사나 연동 이슈가 발생해도 서비스 오픈에는 지장이 없습니다.
3.  **현실적 운영:** 기존에도 계좌이체로 운영하셨으므로, 운영자님에게 익숙한 방식부터 시스템화하는 것이 안전합니다.
4.  **피드백 확보:** 알림(M3)이나 소속감(M4) 기능을 개발하는 동안, 이미 회원들은 M2 기능을 사용하며 피드백을 줄 수 있습니다.

## 3. 상세 재구성 제안

### ✅ M1: 프로젝트 기반 구축 (변동 없음)
- 인프라, 인증, 기본 UI

### ✅ M2: 핵심 신청 흐름 (계좌이체 중심)
- **목표:** "무통장 입금으로 신청 받기"
- **기능:**
  - 모임 신청 (계좌이체 전용)
  - 입금자명 입력, 클립보드 복사
  - "입금했습니다" 버튼 → 정원 예약
  - 대기 신청 기능
  - 운영자 입금 확인/취소 (간이 대시보드)
- **결과:** **🚀 Beta 서비스 오픈 가능 (지인/일부 회원 대상)**

### ✅ M3: 알림 시스템 (변동 없음)
- **목표:** "운영자 수동 알림 해방"
- **기능:** 자동 리마인드, 입금 확인 알림, 48시간 미입금 알림

### ✅ M4: 결제 확장 및 소속감
- **목표:** "편리한 결제와 재미"
- **기능:**
  - **포트원 PG 연동 (카카오페이/토스)**
  - 자동 환불 시스템
  - 칭찬/배지/후기 (소속감 기능 일부)

### ✅ M5: 운영 고도화 및 소속감 완성
- **목표:** "운영 완전 자동화"
- **기능:**
  - 운영자 대시보드 고도화 (통계)
  - 권한 관리, 알림 템플릿 관리
  - 내 책장, 마이페이지 고도화

### ✅ M6: 정식 출시 (변동 없음)
- 신규 회원 유입, 후킹페이지

## 4. 제안에 따른 의존성 및 리스크 검토

**Q: 계좌이체만 먼저 하면 환불은?**  
A: M2 단계에서는 운영자가 계좌로 직접 송금해주고 시스템에서 "환불완료" 버튼을 누르는 수동 방식으로 처리합니다. M4에서 PG가 도입되면 자동 환불도 가능해집니다.

**Q: 알림(M3) 없이 M2 오픈해도 되나?**  
A: 네, 초기 소수 인원(Beta)이라면 운영자가 기존처럼 단톡방/문자로 알림을 보조할 수 있습니다. 대신 M3를 빠르게 이어서 개발해야 합니다.

**Q: M2에 운영자 기능이 일부 필요한데?**  
A: 네, M5에 있던 "입금 확인" 기능만 M2로 가져옵니다. 통계 등 복잡한 대시보드는 M5에 그대로 둡니다.
