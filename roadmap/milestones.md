# 지독해 웹서비스 MVP - Milestones

---

**문서 버전:** 3.1
**작성일:** 2026-01-29 (업데이트)
**관련 문서:** 서비스 개요 v1.3 / PRD v1.8 / 기술 스택 v1.2 / 시스템 구조 v1.6

---

> ⭐ **현재 활성 범위: M1~M6 (MVP)**
> 
> M7 이후 확장 기능(M7~M18)은 MVP 정식 출시 후 별도 로드맵으로 관리됩니다.
> 이 문서에 포함된 M7~M18 내용은 **참고용**으로만 활용하세요.

---

## 1. 개요

이 문서는 지독해 웹서비스 MVP 개발을 위한 최상위 전략인 Milestones를 정의합니다.

**개발 방식:** AI 코딩 에이전트 활용 (Cursor, Claude Code 등)  
**예상 총 기간:** 약 12~14주 (3~4개월)  
**목표:** MVP 완성 후 정식 출시

---

## 3. Milestone 상세

> 📋 각 Milestone의 상세 Work Package는 [work-packages/](./work-packages/) 폴더를 참조하세요.

### M1: 프로젝트 기반 구축

**목표:** 개발 인프라 완성, 기본 화면 동작 확인  
**📋 Work Package:** [WP-M1-기반구축.md](./work-packages/WP-M1-기반구축.md)

| 항목 | 내용 |
|------|------|
| **기간** | 1~2주 |
| **선행 조건** | 없음 (시작점) |
| **핵심 가치** | 쉬운 연결 (기반) |

#### 산출물

| 구분 | 산출물 |
|------|--------|
| **인프라** | Next.js 14 프로젝트 초기화 |
| | Supabase 프로젝트 생성 및 연결 |
| | Vercel 배포 파이프라인 구축 |
| | 환경 변수 설정 (dev/prod 분리) |
| **디자인 시스템** | 8px 그리드 시스템 설정 |
| | Pretendard + Noto Serif KR 폰트 로드 |
| | Framer Motion 설치 및 기본 설정 |
| | 공통 애니메이션 variants 정의 |
| **인증** | 이메일/비밀번호 회원가입 |
| | 이메일/비밀번호 로그인 |
| | 로그아웃 |
| **데이터베이스** | 핵심 테이블 생성 (users, meetings, registrations, refund_policies) |
| | Row Level Security 기본 설정 |
| **UI** | 홈 화면 레이아웃 |
| | 모임 카드 컴포넌트 (컴팩트 디자인 + 호버/클릭 애니메이션) |
| | 카드 리스트 Stagger 등장 효과 |
| | 마감 임박 뱃지 Pulse 애니메이션 |
| | 모임 상세 페이지 (정보 표시만) |
| | 마이페이지 기본 (내 정보) |
| | 모바일 반응형 레이아웃 |
| **운영자** | 모임 생성/수정/삭제 기능 |

#### 검증 포인트

- [ ] 회원가입 → 로그인 → 로그아웃 정상 동작
- [ ] 모임 목록이 홈 화면에 컴팩트 카드로 표시됨
- [ ] 홈 화면 진입 시 카드가 순차적으로 나타남 (Stagger)
- [ ] 모임 카드 호버 시 미세하게 떠오름 (Y축 -4px)
- [ ] 마감 임박 뱃지가 부드럽게 깜빡임 (Pulse)
- [ ] Pretendard/Noto Serif KR 폰트가 적용됨
- [ ] 모임 상세 페이지에서 정보 확인 가능
- [ ] 모바일에서 UI가 깨지지 않음
- [ ] Vercel 자동 배포 동작

#### 의존성 다이어그램

```
없음 (시작점)
    ↓
[M1 완료] → M2 시작 가능
```

### M2: 계좌이체 신청 & Beta 오픈

**목표:** **"무통장 입금으로 신청 받기"** (조기 오픈 달성)  
**📋 Work Package:** [WP-M2-계좌이체신청.md](./work-packages/WP-M2-계좌이체신청.md)

| 항목 | 내용 |
|------|------|
| **기간** | 2주 |
| **선행 조건** | M1 완료 |
| **핵심 가치** | 쉬운 연결 (신청 시작) |
| **결과** | **🚀 Beta 서비스 오픈 (지인/기존 회원 대상)** |

#### 산출물

| 구분 | 산출물 |
|------|--------|
| **인증 확장** | 카카오 로그인 연동 (Supabase Auth) |
| **계좌이체** | 계좌 정보 표시 UI (클립보드 복사) |
| | 입금자명 입력 ("MMDD_이름") |
| | "입금했습니다" 버튼 → 정원 예약 |
| | 48시간 타이머 및 미입금 만료 로직 (기본) |
| **입금 관리** | 운영자용 "입금 확인" 목록 (간이) |
| | 입금 확인 버튼 → 참가 확정 처리 |
| | 입금 취소/환불 처리 (수동 상태 변경) |
| **환불(수동)** | 취소 요청 UI + 환불 계좌 입력 폼 |
| | 취소 사유 입력 |
| **대기** | 정원 초과 시 대기 등록 |
| | 대기자 목록 관리 |
| | 취소 발생 시 수동 알림 (알림톡은 M3) |
| **UI** | 신청 버튼 활성화 |
| | 참가 인원 표시 ("O명 참여", 정원 숨김) |
| | 마감 임박/마감 상태 표시 |
| | 마이페이지 신청 내역 (입금대기/완료/취소 상태) |

#### 검증 포인트

- [ ] 계좌이체 신청 시 정원이 즉시 예약됨
- [ ] 운영자가 "입금 확인"을 누르면 마이페이지에서 "신청 완료"로 변경됨
- [ ] 카카오 로그인 정상 동작
- [ ] 취소 시 환불 계좌 입력 폼이 표시됨
- [ ] 정원 초과 시 대기 등록 가능
- [ ] **실제 회원에게 링크를 줘서 신청을 받을 수 있음**

> **Strategy:** PG 심사를 기다리지 않고, 가장 빠른 방법인 계좌이체로 먼저 오픈하여 신청을 받습니다.

### M3: 알림 시스템

**목표:** 회원이 모임을 잊지 않게, 자동 리마인드 시스템 구축  
**📋 Work Package:** [WP-M3-알림시스템.md](./work-packages/WP-M3-알림시스템.md)

| 항목 | 내용 |
|------|------|
| **기간** | 1~2주 |
| **선행 조건** | M2 완료 |
| **핵심 가치** | 잊지 않게 |

#### 산출물

| 구분 | 산출물 |
|------|--------|
| **알림 인프라** | 솔라피 알림톡 연동 |
| | 알림 서비스 추상화 레이어 (추후 교체 용이) |
| | 알림 발송 기록 저장 (notification_logs) |
| **자동 알림** | 모임 리마인드 (3일 전, 1일 전, 당일) |
| | 대기자 자리 발생 알림 |
| | 응답 대기 시간 로직 (24시간/6시간/2시간) |
| | 월말 참여 독려 알림 |
| | 세그먼트별 리마인드 (온보딩 이탈 위험, 휴면 위험, 자격 만료 임박) |
| **스케줄링** | Vercel Cron 또는 Supabase pg_cron 설정 |
| | 대기자 응답 시간 초과 체크 |
| **운영자** | 수동 알림 발송 (공지용) |

#### 검증 포인트

- [ ] 모임 3일 전 리마인드 알림 발송됨
- [ ] 대기자에게 자리 발생 알림 발송됨
- [ ] 응답 대기 시간 초과 시 다음 대기자에게 알림
- [ ] 세그먼트별 리마인드 발송 (온보딩 이탈 위험, 휴면 위험, 자격 만료 임박)
- [ ] 운영자가 수동으로 알림 발송 가능
- [ ] 알림 발송 기록이 저장됨

#### 의존성 다이어그램

```
[M2 완료]
    ↓
[M3 완료] → M4와 병행 가능
    ↓
    → M5의 알림 템플릿 관리에 필요
```

**관련 문서:** 하드코딩 분석 결과 / 백오피스 요구사항 분석

**핵심 Pain Point:**
- 계좌이체 환불 시 수동 송금 필요 (은행 앱 열어서 직접 이체)
- "왜 신청이 안 되나요?" 문의 시 자격 상태 파악 어려움
- 하드코딩된 운영 파라미터 변경 시 배포 필요
- 입금 확인, 노쇼 처리 등 반복 작업 비효율

**성공 기준:** 일일 운영 업무를 **30분 이내**로 처리 가능

**예상 기간:** 약 6~8주
**선행 조건:** M6 완료 (정식 출시), M5 운영자 도구 기반 활용, M7과 병행 가능
