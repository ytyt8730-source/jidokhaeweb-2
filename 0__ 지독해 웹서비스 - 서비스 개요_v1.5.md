## 1. 서비스명
**지독해 (가칭: 지독해 웹)**

---

## 2. 서비스 한 줄 정의
> 지독해 회원들이 모임과 우아하게 연결되어 있다는 감각을 유지할 수 있게 해주는 **독서 리추얼 플랫폼**

---

## 3. 해결하려는 문제

### 외적 문제 (External)

**회원 관점**
- 카카오톡 일정이 한눈에 안 들어와서 모임 확인이 불편함
- 참가 신청(입금자명 수정 → 계좌이체 → 댓글)이 번거로움
- 취소/환불 요청 시 개인 연락 → 확인 → 처리까지 시간이 오래 걸림
- 바쁜 일상에서 신청해놓고 까먹거나, 뭘 신청했는지 헷갈림
- 리마인드가 없으면 모임 존재 자체를 잊어버림

**운영자 관점**
- 환불 요청 시 입금 내역을 일일이 찾아서 확인해야 함
- 이번 달 수입, 공간대여비, 재참여율 등 현황 파악이 어려움
- 정보가 카톡, 구글시트, 은행앱 등에 흩어져 있어 기억에 의존해야 함

### 내적 문제 (Internal)

- 새 모임에 나가는 게 어색하고 부담스러움
- 꾸준히 참여할 자신이 없음
- 신청해놓고 빠지면 민망함
- 진지한 대화를 나눌 곳이 없다는 막막함

### 철학적 문제 (Philosophical)

> **"바쁜 어른도 새로운 인연을 만들고 소속될 곳이 있어야 한다."**

### 빌런 (Villain)

> 바쁘다는 핑계로 사람을 만나지 않게 만드는 **일상의 관성**

---

## 4. 본질

**표면:** 복잡한 프로세스를 간소화하고 싶다.

**본질:** 바쁜 일상 속에서 책을 매개로 **새로운 사람들과 연결**되고, **어딘가에 속해 있다는 감각**을 느끼게 해주고 싶다.

> "바쁜 현생을 사는 회원들이 **Frictionless(마찰 없는)** 경험으로 자연스럽게 스며들고, 지적인 충만함을 느끼게 한다."

## 5. 핵심 가치

| 가치 | 키워드 | 설명 |
|------|--------|------|
| **쉬운 연결** | Frictionless | 페이지 이동 없는 **Bottom Sheet**와 **Sticky CTA**로 물 흐르듯 신청 |
| **분위기** | Atmosphere | Atmospheric Blur와 Motion으로 만드는 공간감 있는 경험 |
| **소속감** | Pride | `<Trophy />`와 Glow 효과로 연출되는 고급스러운 성취 경험 |
| **신뢰** | Trust | `<KongIcon />` 기반의 투명하고 직관적인 화폐 시스템 |
| **자연스러운 확산** | Spread | 공유하기와 같이가기(찔러보기)로 회원이 자연스럽게 새로운 연결을 만드는 경험 |

## 6. 타겟 사용자

**주 사용자: 지독해 기존 회원**
- 정기모임에 참여하거나 참여하려는 회원
- 바쁜 일상 속에서 독서모임을 유지하고 싶은 사람

**부 사용자: 신규 회원**
- 지독해를 처음 알게 된 사람
- 후킹 랜딩페이지를 통해 유입되어 첫 정기모임을 신청하는 사람

**운영자: 영탁님(단무지)**
- 모임 운영, 정산, 회원 관리를 담당
- 다른 운영진에게 일부 권한 위임 가능

---

## 7. 핵심 규칙 (서비스에 반영)

**No-Emoji 정책**
- 서비스 내 모든 이모지 사용을 금지하며, Lucide 아이콘과 커스텀 SVG로 대체한다.

**참여 자격**
- 모든 모임 참여의 전제조건 = 정기모임 참여 이력
- 6개월 이내 정기모임 참여 기록 없으면 다른 모임 참여 불가

**환불 규정**
- 모임 유형별 환불 규정 적용 (상세: PRD 섹션 5 "환불 규정 기본값" 참조)

**화폐 시스템**
- 참가비 단위: **"콩"**
- 반드시 `<KongIcon />`과 함께 표기 (예: `[KongIcon] 10,000콩`)

## 8. 주요 기능

> **MVP 원칙:** "이 서비스가 필요한가?"를 가장 빠르게 검증하는 것이 목적이다. 완성된 경험이 아닌, 핵심 흐름의 검증에 집중한다. Alpha 출시까지 5~6주를 목표로 한다.

### True MVP (Phase 0~2, 5~6주)

**일정 경험 (Visual Experience)**
- **Staggered Reveal:** 페이지 진입 시 일정 카드가 100ms 간격으로 순차적으로 떠오르는 애니메이션
- **One-Page Flow:** 일정 클릭 시 페이지 이동 없이 **Bottom Sheet**가 부드럽게 올라옴
- **Atmospheric Card:** 모임 커버 이미지를 배경 블러 처리하여 깊이감 조성
- 정보 표기: `<Calendar />` 일시, `<MapPin />` 장소, `<Users />` 참가 인원 (정원은 숨김)

**신청/결제 (Frictionless)**
- **Sticky CTA:** Bottom Sheet 하단에 고정된 버튼으로 즉시 결제
- 버튼 텍스트: `[KongIcon] 10,000콩으로 신청하기`
- **간편결제 전용** (카카오페이, 토스페이먼츠) — 계좌이체는 Growth Phase에서 추가
- 대기 신청 (자리 나면 알림 → 그때 결제)

**취소/환불 (Self-Service)**
- 셀프 취소 + 자동 환불 (규정에 따라)
- 취소 사유 입력 필수 (운영 개선용)
- 대기자 자동 승격 (취소 발생 → 순차 알림)

**리마인드 (Connection)**
- 자동 리마인드 알림톡 (모임 3일 전, 1일 전, 당일)
- **가치 제안 중심 메시지:** "안 했다"가 아니라 "얻을 수 있는 것"을 전달
- **No-Emoji:** 알림톡 메시지 내 이모지 사용 금지, 텍스트 위주로 정중하게 작성
- 운영자 수동 알림 발송 (공지, 이벤트 안내 등)

**공유하기 (Spread)**
- 모임 공유: Bottom Sheet 내 공유 버튼 → Web Share API 또는 링크 복사
- 신청 확정 후 공유 유도: Toast에 [공유하기] 버튼 포함
- 공유 URL에 레퍼럴 파라미터 포함 (ref={user_id})

**같이가기 — 찔러보기 (Spread)**
- 신청 완료 상태에서 다른 회원에게 "같이 가자" 알림 발송
- 회원 이름 검색 → 최대 3명 선택 → 알림톡 발송 (No-Emoji)
- 한 모임에 같은 사람에게 1회만 발송 가능
- 수신 거부 옵션 제공 (마이페이지 설정)
- 누가 찔렀는지는 비공개

**운영자 기능 (Admin — MVP)**
- 모임 생성/수정/삭제
- 알림톡 수동 발송 (공지, 이벤트 안내 등)

---

### Growth Phase (MVP 검증 후)

MVP Alpha/Beta에서 "회원들이 웹 신청을 선호한다"가 확인된 후 순차 진행한다.

- 배지 시스템 (Confetti Canvas + 3D 카드 회전)
- 칭찬하기 (Aha Moment 트리거)
- 내 책장 (`<BookOpen />` + 한 문장 기록)
- 후기 남기기
- 건의하기
- 5단계 온보딩 플로우 (신규 회원)
- 후킹 랜딩페이지 (브랜드 스토리, 분위기, 회원 후기)

- 계좌이체 결제 옵션
- 세그먼트별 맞춤 리마인드 + 알림 중단 정책
- 알림 템플릿 관리 (운영자)
- 대시보드 (참가 현황, 수입, 환불, 재참여율, 선행 지표)
- 운영자 권한 관리
- 요청함 (문의 확인 + 답변)
- 홈 화면 콘텐츠 배너 관리

---

### 추후 기능 (2차 이후)

- 결제 수단 미리 등록 (원클릭 결제)
- 대기 신청 시 선결제 → 자동 환불
- 구글/애플 캘린더 자동 추가
- 정기모임 개설 요청 (회원들이 원하는 지역/요일 투표)
- 다른 회원 책장 구경
- 책 등록 배지 (10권, 50권 등)
- 참여 리포트 (월말/연말 "올해 OO회 참여")
- 클럽장 콘텐츠 정산 자동화 (80/20 자동 계산)
- 토론모임, 지글챌, 번개모임 전용 기능
- 진행자 시스템
- 오늘의 지독해 (소식 피드)
- 레퍼럴/초대 시스템 고도화

## 9. 사용자 경험 흐름 (UX Flow)

**기존 회원 (3-Click Flow)**

웹 접속
→ 홈: Bento Grid 대시보드 + Staggered Animation으로 일정 카드 등장
→ [TAP] 일정 카드 클릭
→ [ACTION] Bottom Sheet 오픈 (페이지 이동 없음)
  - Atmospheric Blur 배경
  - 모임 상세 정보 표시
→ [TAP] 하단 Sticky CTA "[KongIcon] 10,000콩으로 신청하기"
→ [ACTION] 포트원 결제 창 오버레이
→ 결제 완료 → Bottom Sheet 닫히며 "참여 확정" Toast 메시지
→ Toast 내 [공유하기] 버튼 노출 — "함께할 사람에게 알려보세요"
→ (선택) [같이가기] 버튼 탭 → 회원 검색 → 찔러보기 발송
→ (모임 3일 전, 1일 전, 당일) 리마인드 알림톡
→ 모임 참여
→ (Growth Phase) 칭찬하기 유도 알림

**신규 회원**

웹 접속 → 일정 화면
→ [TAP] 일정 카드 클릭
→ [ACTION] Bottom Sheet 내 안내 모달 오버레이:
  "지독해가 어떤 곳인지 먼저 보시는 걸 추천해요."
  [Link] 브랜드 스토리 보러가기
  [Button] 바로 신청하기
→ (브랜드 스토리 선택 시) Bento Grid 후킹페이지 확인 → 신청
→ (바로 신청 선택 시) Bottom Sheet 내 결제 진행
→ 첫 모임 전날 환영 알림 (준비물, 장소 안내)
→ 첫 모임 참여
→ (다음 날 오전 7시) "어제 함께해서 좋았습니다. 인상 깊었던 분에게 한마디 남겨보실래요?"

**취소/환불**

마이페이지 → 신청 내역 → 취소하기
→ [ACTION] 마음 돌리기 Bottom Sheet 오픈 (함께 신청한 회원, 모임 임박 등 표시)
→ [TAP] 취소 사유 선택 (필수)
→ [ACTION] 환불 규정 안내 + 동의
→ 최종 확인 후 자동 환불
→ 대기자 있으면 자동 승격 알림

※ 취소는 신청보다 마찰을 높여 충동적 취소 방지 (상세: PRD 섹션 6)

## 10. 기대 효과

### 회원의 변화 (Before → After)

| Before | After |
|--------|-------|
| 모임 확인/신청이 번거로워서 미루게 됨 | 3번 탭으로 신청 완료, 참여 허들이 낮아짐 |
| 신청해놓고 까먹어서 노쇼 | 리마인드로 모임을 잊지 않음 |
| 첫 모임 후 어색해서 안 감 | 칭찬 주고받으며 연결감 형성 |
| 어디에도 속하지 않은 느낌 | **"수요일엔 지독해 가는 날" — 나도 여기 사람이 되었다** |
| 신규 회원 유입 경로 없음 | 공유하기 + 찔러보기로 기존 회원이 자연스럽게 새 회원을 유입 |
| 혼자 가기 부담스러움 | 찔러보기로 "같이 가는 사람"이 생겨 첫 참여 부담 감소 |

### 운영자의 변화

- 정산/환불 처리 시간 대폭 단축
- 모든 정보가 한 곳에 → 기억에 의존하지 않아도 됨
- 재참여율 등 데이터 기반 운영 가능
- 알림 문구/시점을 유연하게 조정 가능

### 지독해 전체

- 회원 이탈 감소, 재참여율 증가
- 기존 회원의 입소문을 통한 자연스러운 신규 유입 (Acquisition 루프)
- 브랜드 이미지 강화 ("체계적이고 신뢰할 수 있는 모임")
- "뻔한 AI 디자인"을 탈피한 독창적 브랜드 경험

## 11. 브랜드 톤 (Brand Voice)

### 시각적 톤

힙하고 에너지 넘치는 독서 라운지 느낌의 단일 컬러 체계를 사용한다.

| 요소 | 값 | 비고 |
|------|-----|------|
| Primary | Cobalt Blue (#0047FF) | 주요 액션, 링크 |
| Accent | Acid Lime (#CCFF00) | 배경 전용, 텍스트 사용 금지 |
| Background | #F8FAFC | 클린 화이트 |
| 헤드라인 | Outfit / Noto Sans KR | 고딕체(Sans) |
| 본문 | Noto Sans KR | - |

> **참고:** 디자인 시스템은 추후 새로 작성 예정이다. 위 값은 현재 기준이며 변경될 수 있다.

### 디자인 원칙

- **No-Emoji:** 모든 이모지 사용 금지. Lucide React 아이콘으로 대체.
- **One-Page:** 페이지 이동 최소화. Bottom Sheet 패턴 활용.
- **3-Click Rule:** 신청 완료까지 3번의 탭 이내.

### 창업자 스토리 (Guide Voice)

> **"혼자 읽으니 지루하지 않으세요?"**
> 
> 저도 그랬어요.
> 책 사놓고 안 읽은 지 몇 달째,
> 진지한 대화 나눌 곳도 없고.
> 
> 그래서 4명이 모여봤습니다.
> "이별"에 대해 이야기했는데,
> 다들 이별한 지 얼마 안 됐더라고요.
> **그런데 우리는 웃고 있었어요.**
> 
> 혼자가 아니면 다르더라고요.
> 
> 그 모임이 3년째 이어지고 있고,
> 지금은 매주 경주와 포항에서 만나고 있습니다.
> 
> **당신도 함께할 수 있어요.**

### 회원 후기 (권위의 간접적 증거)

> "처음엔 어색했는데, 이제는 수요일이 기다려져요."
> — 함께한 지 180일째 회원

> "책 사놓고 안 읽던 제가 올해 12권 읽었어요."
> — 함께한 지 365일째 회원

> "여기선 진지한 얘기해도 귓등으로 안 들어요."
> — 함께한 지 90일째 회원

## 12. 브랜드스크립트 (SB7)

도널드 밀러의 StoryBrand 프레임워크 기반.

### Character (캐릭터)

바쁜 일상 속에서 책을 매개로 **새로운 사람들과 연결**되고, **어딘가에 속해 있다는 감각**을 원하는 사람

### Problem (문제)

| 층위 | 내용 |
|------|------|
| **외적** | 입금자명 변경 → 계좌이체 → 댓글 → 확인 대기 (4단계) |
| **내적** | 새 모임 나가기 어색함, 꾸준히 참여할 자신 없음, 신청해놓고 빠지면 민망함 |
| **철학적** | 바쁜 어른도 새로운 인연을 만들고 소속될 곳이 있어야 함 |
| **빌런** | 바쁘다는 핑계로 사람을 만나지 않게 만드는 일상의 관성 |

### Guide (가이드)

| 요소 | 내용 |
|------|------|
| **공감** | "저도 혼자 읽으니 지루했어요. 그래서 4명이 모여 시작했습니다." |
| **권위** | 3년째 이어지는 모임 + 회원 후기 |

### Plan (계획)

① 마음에 드는 모임 고르기 → ② 3번 탭으로 신청하기 → ③ 리마인드 받고 참여하기

### CTA (행동 촉구)

| 유형 | 메시지 |
|------|--------|
| 직접적 | `[KongIcon] 10,000콩으로 신청하기` |
| 전환적 | "3분 만에 지독해 알아보기" / "이번 달 일정 보기" |

### Failure (실패)

- 또 한 달이 그냥 지나간다
- 아는 얼굴들이 점점 낯설어진다
- 자격 만료 시 처음부터 다시 시작해야 한다

### Success (성공)

> **"수요일엔 지독해 가는 날"**
> — 나도 여기 사람이 되었다
